## 📊 MULTI-SOURCE ARCHITECTURE

**We never rely on one data source. We integrate multiple authoritative sources:**
```
SUNLIGHT Data Sources:
├─ USAspending.gov (federal contracts, price data)
├─ OpenSecrets.org (political donations, influence mapping)
├─ FPDS (contract modifications, bid competition, amendments)
├─ Minneapolis.gov (local contracts for case studies)
├─ PACER (validated fraud cases for methodology testing)
├─ FOIA requests (deep investigation evidence, internal docs)
└─ [Future] State/local databases (scalable to any jurisdiction)
```

**Why multiple sources:**
1. **Resilience:** If one API fails, we continue
2. **Validation:** Cross-reference data for accuracy
3. **Completeness:** Each source adds a missing piece
4. **Credibility:** More sources = more authoritative

## 🎯 DETECTION PHILOSOPHY

### **Conservative by Design**
- We flag what we CAN prove, not what we suspect
- 0 RED cases with honesty > 50 RED cases with doubt
- Missing data = admission of limitation, not guess-work

### **Evidence-Based**
- Every flag requires multiple indicators
- Every indicator cites source data
- Every conclusion shows reasoning

### **Legally Grounded**
- Align with DOJ prosecution standards
- Reference actual prosecuted cases
- Cite specific legal frameworks (False Claims Act, Anti-Kickback, etc.)

### **Statistically Rigorous**
- Proper baselines (apples-to-apples)
- Confidence intervals documented
- Sample sizes disclosed
- False positive rates calculated

## 🔍 TRANSPARENCY REQUIREMENTS

### **Every Analysis Must Include:**

1. **Data Sources**
   - Where each data point came from
   - When it was collected
   - How it was verified

2. **Methodology**
   - Which algorithms were used
   - Why those algorithms
   - Alternative approaches considered

3. **Calculations**
   - Step-by-step math shown
   - Statistical significance
   - Confidence levels

4. **Evidence**
   - Primary sources linked
   - Supporting documents
   - Comparable cases

5. **Limitations**
   - What data is missing
   - What we can't conclude
   - Where uncertainty exists

6. **Reasoning**
   - Why this pattern is suspicious
   - What would make it normal
   - Legal basis for concern

## 🚫 WHAT WE NEVER DO

**Never:**
- ❌ Claim certainty without complete evidence
- ❌ Rely on a single data source
- ❌ Hide methodology or algorithms
- ❌ Lower standards to get flashy results
- ❌ Name individuals without legal process
- ❌ Assume guilt instead of flagging patterns
- ❌ Ignore context or legitimate exceptions

**Always:**
- ✅ Show complete evidence trail
- ✅ Admit what we don't know
- ✅ Invite expert scrutiny
- ✅ Provide source links
- ✅ Document limitations
- ✅ Recommend investigation, not prosecution
- ✅ Open to correction and improvement
